## 一、系统定位

Free-Hire 是一套可自托管的招聘工作台，主流程是：

1. 创建岗位
2. 上传或拉取简历
3. 在人才库沉淀候选人档案
4. AI 生成题集 / 题库
5. 发起候选人远程面试
6. 自动生成报告与分析结果
7. 在后台完成团队、审计、SSO 与 License 管理

核心模块：

| 模块 | 作用 |
|---|---|
| **简历库** | 简历上传、解析、下载、修正信息 |
| **人才库** | 聚合候选人简历、匹配、面试、标签、分组、黑名单、备注 |
| **岗位** | JD 管理、状态流转、匹配候选人 |
| **岗位治理** | 能力模型、JD 优化、审批、版本记录、协作备注 |
| **面试题集** | 从简历一键生成面试题 |
| **题库** | 租户级可复用问题池 |
| **面试** | HR 发起远程面试，候选人异步答题 |
| **报告** | KPI、来源分布、推荐率、岗位填充、热门技能 |
| **数据分析** | 趋势、漏斗、评分分布、高频题目分析 |
| **系统设置** | LLM、IMAP、SMTP、语音、License、团队、SSO、审计 |

## 二、角色与权限

系统内置 5 类角色：

| 角色 | 主要职责 |
|---|---|
| `admin` | 全权限，负责系统设置、团队、License、审计、SSO |
| `hr` | 日常招聘作业：岗位、简历、题集、面试、报告 |
| `interviewer` | 参与面试与查看结果 |
| `hiring_manager` | 查看职责范围内岗位与报告，并可编辑岗位 |
| `viewer` | 只读查看 |

补充说明：

- 菜单会按权限动态显示
- 后端接口会再次按权限和组织范围校验
- 非 `admin` 默认可能只看到自己组织范围内的数据
- `viewer` 无法创建、编辑、删除业务数据

## 三、首次部署（IT）

> 更完整的安装步骤见 [`offline-install.md`](offline-install.md)。

### 3.1 解压并安装

```bash
tar xzf free-hire-{VERSION}.tar.gz
cd free-hire-{VERSION}/
./install.sh
```

首次执行会提示你编辑 `.env`。

### 3.2 至少修改这些配置

| 字段 | 说明 |
|---|---|
| `JWT_SECRET` | 用 `openssl rand -hex 32` 生成强随机串 |
| `BOOTSTRAP_ADMIN_EMAIL` | 首位管理员邮箱 |
| `BOOTSTRAP_ADMIN_PASSWORD` | 首位管理员强密码 |
| `MACHINE_FINGERPRINT_OVERRIDE` | 用 `openssl rand -hex 16` 生成；用于申请 license |

然后再次执行：

```bash
./install.sh
```

成功后会看到类似输出：

```text
✓ Free-Hire 已启动
访问: http://localhost
API 文档: http://localhost/api/docs
```

### 3.3 申请并导入 License

查看机器指纹：

```bash
grep MACHINE_FINGERPRINT_OVERRIDE .env
```

把该值发给供应商，获取 `.lic` 文件后：

1. 打开 `http://localhost`
2. 用 `BOOTSTRAP_ADMIN_*` 登录
3. 进入 **系统 → License 设置**
4. 上传 `.lic`

如果暂时没有 `.lic`，系统仍可进入 30 天试用期。

### 3.4 首次登录后建议的初始化顺序

建议管理员第一次登录按这个顺序做：

1. **License 设置**：确认试用 / 激活状态
2. **LLM 配置**：配置真实模型或内网网关
3. **SMTP 发件**：让面试邀请邮件可直接发送
4. **邮箱拉取**：配置 IMAP 自动收简历
5. **语音 STT/TTS**：如果要用语音面试
6. **团队**：创建 HR / 面试官 / 用人经理账号
7. **SSO 接入**：如果需要企业统一登录

## 四、管理员初始化配置

### 4.1 LLM 配置

进入 **系统 → LLM 配置**。

常见配置方式：

- **内网模型网关**：Base URL 指向公司内网的 OpenAI 兼容服务
- **公有云模型**：OpenAI、Azure OpenAI、通义、火山等
- **自建服务**：vLLM、Ollama、代理网关

注意：

- 若接入外部公网模型，候选人简历和问答内容会发送到该模型供应商
- 若企业要求数据不出网，请使用内网模型或内网代理网关

### 4.2 IMAP 邮箱拉取

进入 **系统 → 邮箱拉取**。

可配置招聘邮箱，让系统定时自动抓取附件简历入库。

基本步骤：

1. 添加邮箱账户
2. 填写 IMAP 服务器、端口、账号、密码或 app password
3. 选择收件箱文件夹
4. 启用账户
5. 测试连接或手动触发一次同步

### 4.3 SMTP 发件

进入 **系统 → SMTP 发件**。

配置后，HR 在发起远程面试时可直接选择：

- 仅邮件
- 仅链接
- 邮件 + 链接

### 4.4 语音 STT/TTS

进入 **系统 → 语音 STT/TTS**。

语音面试依赖这部分配置：

- STT：语音转文字
- TTS：文字转语音

即使语音 Provider 已配置，实际发起语音面试仍受 `interview.voice` License 功能位控制。

### 4.5 团队管理

进入 **系统 → 团队**。

常见操作：

- 邀请成员
- 修改角色
- 禁用 / 启用账号
- 重置密码
- 删除账号

注意：

- 初始密码只会在弹窗中明文显示一次
- 禁用账号后，新的登录会被拒绝，已签发 token 也会失效
- 系统有自我保护，用户不能删除或修改自己当前账号的关键身份

### 4.6 SSO 接入

进入 **系统 → SSO 接入**，仅 `admin` 可见。

当前支持 OIDC / OAuth2 风格配置，主要字段包括：

- 登录入口显示名
- Issuer / Authorize / Token / UserInfo URL
- Client ID / Client Secret
- Redirect URI
- 自动建号开关
- 默认角色
- 默认组织
- Email / Name / Role / Org claim 映射
- 角色 / 组织 JSON 映射规则

启用后：

- 登录页会出现“企业统一登录”入口
- 本地账号密码登录仍保留为兜底

### 4.7 审计中心

进入 **系统 → 审计中心**，仅 `admin` 可见。

可按对象、动作、结果、操作人、对象 ID 进行检索。典型留痕包括：

- 简历上传、下载、删除
- 报告导出
- 评分覆核
- 用户、组织、角色变更
- 岗位治理动作
- 面试相关关键操作

## 五、HR 日常使用流程

### 5.1 创建岗位

进入 **岗位**：

1. 点击“新建岗位”
2. 填写标题、级别、关键技能、岗位描述
3. 创建后可继续维护状态与详情

岗位状态说明：

- `open`：招聘中
- `paused`：暂停
- `closed`：关闭 / 归档

### 5.1.1 岗位治理

在岗位详情页可以看到治理相关能力：

- **能力模型**：维护结构化能力项、权重、是否必需
- **AI 生成能力模型**：根据 JD 自动生成建议
- **JD 优化建议**：生成建议与改写版本，不会直接覆盖原文
- **审批流**：提交审批、通过、驳回、重开、关闭
- **版本记录**：每次关键变更会形成版本快照
- **协作备注**：供 HR、用人经理协作

适用场景：

- 多人协作审核 JD
- 标准化岗位能力模型
- 留存岗位历史变更依据

### 5.2 上传与管理简历

进入 **简历库**。

### 手动上传

支持格式：

- PDF
- DOCX
- TXT

上传后，简历会经历这些状态：

- `pending`：已入库，等待 worker 处理
- `parsing`：正在解析
- `done`：解析完成
- `failed`：解析失败

在简历详情抽屉中可查看：

- 候选人姓名、邮箱、手机号
- 解析出的技能
- 原始文本
- 关联题集
- 关联岗位匹配

### 邮箱自动入库

若管理员已配置 IMAP，系统会自动抓取招聘邮箱里的附件简历并入库。

### 简历修改与下载

HR 可以在简历详情中：

- 修正候选人邮箱 / 手机
- 下载原始简历文件
- 删除简历

### 5.3 人才库运营

进入 **人才库**。

人才库以候选人为主视角，自动聚合：

- 简历版本
- 面试历史
- 岗位匹配
- 标签
- 分组
- 黑名单
- 最近备注
- 时间线

常见用法：

- 给候选人打标签，例如 `Java`、`高潜`、`待复联`
- 建立分组，例如“2026 届校招”“前端储备池”
- 加入黑名单并记录原因
- 写备注沉淀沟通记录
- 在详情页查看候选人完整轨迹

### 5.4 生成面试题集

有两种入口：

- **简历库 → 简历详情 → 生成面试题**
- **面试题集** 页面查看历史生成结果

常见配置项：

- 难度：初级 / 中级 / 高级 / 专家
- 题量：5 / 10 / 15
- 题型：技术深度 / 项目复盘 / 场景排查 / 系统设计 / 软技能
- 可指定关联岗位

生成结果包含：

- 题干
- 答题要点
- 题型
- 难度
- 追问

适合场景：

- HR / 面试官准备真人面试问题
- 面试前快速形成候选人定制题单

### 5.5 管理题库

进入 **题库**。

题库是租户级可复用问题池，支持：

- 手动新增单题
- 编辑、删除题目
- 按分类、题型、难度筛选
- 基于岗位 / 分类批量 AI 生成

适合：

- 建立公司自己的标准题库
- 沉淀高质量面试题
- 淘汰区分度低的问题

### 5.6 岗位匹配

从岗位或简历侧都能触发匹配：

- **岗位详情 → 匹配候选人**
- **简历详情 → 岗位匹配**

系统会生成：

- 0-100 匹配分
- 优势点
- 差距点
- 评价说明

常见用途：

- 快速排序候选人优先级
- 从简历池中筛岗位候选人
- 从岗位视角寻找最适合的候选人

### 5.7 发起远程 AI 面试

进入 **面试** 页面。

当前正式主流程是“候选人远程异步答题”，不是 HR 代答模式。

### 5.7.1 创建面试

1. 点击“发起面试”
2. 选择候选人
3. 选择岗位
4. 配置：
   - 难度
   - 题数
   - 题型
   - 面试形式：文字 / 语音
   - 语音模式下的单题时长
   - 截止时间
   - 送达方式：仅链接 / 仅邮件 / 邮件 + 链接
   - 收件邮箱
5. 生成邀请链接

说明：

- 语音面试要求 `interview.voice` 功能位已开启
- 若候选人简历中有邮箱，系统会自动带入收件邮箱
- 若送达方式包含邮件，但 SMTP 不可用，HR 仍可复制链接手动发送

### 5.7.2 候选人答题流程

候选人打开链接后：

1. 进入独立候选人页面，不需要 HR 后台账号
2. 若候选人有手机号，通常需要输入手机号末 4 位验证
3. 验证通过后进入答题页面
4. 完成全部题目后自动结束
5. HR 侧稍后可查看报告

### 5.7.3 会话与失效规则

候选人答题 session 有这些规则：

| 场景 | 行为 |
|---|---|
| 候选人首次打开链接 | 若启用手机号校验，需要通过验证后才签发答题 session |
| 候选人关闭 tab | 前端 session 丢失，需要重新验证进入 |
| 候选人 6 小时内未答完 | session 过期，但已答题进度保留；重新验证后可继续 |
| HR 重新发送邀请 | 旧链接和已签发 session 立刻失效 |
| HR 取消邀请 | 链接和 session 立刻失效 |
| 候选人中途收到 401 | 前端会跳回 invite 页面，提示重新验证 |

### 5.7.4 查看报告

进入 **面试报告** 页面可查看：

- 综合分
- 推荐结论
- 各维度评分
- 每道题的问答与证据
- 语音信号（若为语音模式）

若 License 启用了 `report.export`，可导出 PDF。

### 5.8 看报告与数据分析

### 报告页

进入 **报告**：

- 候选人总数
- 简历总数
- 进行中岗位
- 面试总数
- 推荐通过率
- 简历来源分布
- 简历解析状态分布
- 面试状态分布
- 岗位填充进度
- 热门技能 Top 10

### 数据分析页

进入 **数据分析**：

- 面试趋势
- 候选人漏斗
- 高频题目分析
- 评分分布

该页更适合管理层或招聘负责人看整体招聘质量与流程损耗。

## 六、运维（IT）

### 6.1 启停

```bash
docker compose down
docker compose --env-file .env up -d
docker compose down -v
```

说明：

- `down`：停服务，保留数据
- `up -d`：后台启动
- `down -v`：同时删除 PG / Redis / 对象存储数据卷

### 6.2 日志

```bash
docker compose logs -f backend
docker compose logs -f worker
docker compose logs -f frontend
docker compose logs --since 1h
```

### 6.3 备份

```bash
cd free-hire-{VERSION}/
./backup.sh
./backup.sh /mnt/nas/freehire
FREEHIRE_KEEP_LAST=7 ./backup.sh /mnt/nas/freehire
```

备份包中通常包括：

- PostgreSQL dump
- 对象存储卷
- 脱敏后的 `.env` 参考副本
- manifest 信息

### 6.4 还原

```bash
cd free-hire-{VERSION}/
./restore.sh /path/to/free-hire-backup-xxxx.tar.gz
```

恢复前脚本会做二次确认。

### 6.5 升级

```bash
cd free-hire-{OLD_VERSION}/
./backup.sh /tmp/

tar xzf free-hire-{NEW_VERSION}.tar.gz
cd free-hire-{NEW_VERSION}/
cp ../free-hire-{OLD_VERSION}/.env .env
./install.sh
```

### 6.6 健康检查

```bash
curl http://localhost/api/healthz
curl http://localhost/api/healthz?detail=1
```

说明：

- `healthz`：看服务存活
- `healthz?detail=1`：看 DB、Redis、Broker、存储等子系统

### 6.7 Prometheus 指标

如需在生产抓取，请在 `.env` 中设置：

```text
EXPOSE_OPERATIONAL_DIAGNOSTICS=true
```

然后抓取：

```text
/api/metrics
```

生产环境应只开放给内网或受控监控网络。

## 七、License 与续期

### 7.1 当前功能位

当前代码中的功能位为：

| 功能位 | 说明 |
|---|---|
| `resume.upload` | 简历上传与解析 |
| `resume.email` | IMAP 邮箱拉取 |
| `interview.text` | 文本面试 |
| `interview.voice` | 语音面试 |
| `match.evaluate` | 简历岗位匹配 |
| `report.export` | 报告导出 |
| `team.multi` | 多 HR 协作 |

### 7.2 版本与配额

| | `community` | `professional` | `enterprise` |
|---|---|---|---|
| 功能位 | 4 项核心功能 | 7 项全开 | 7 项全开 |
| 近 30 天简历数 | 50 | 500 | 无限 |
| HR / 管理员账号数 | 1 | 10 | 无限 |
| 岗位数 | 5 | 无限 | 无限 |

规则：

- 首次部署后默认有 30 天试用期
- 试用期等同 `professional`
- 到期后自动回落到 `community`
- 已有数据始终可查看，只限制新增

### 7.3 查看当前状态

进入 **系统 → License 设置** 可看到：

- 当前 edition
- 是否试用 / 已激活 / 已过期
- 剩余天数
- 机器指纹
- 功能位启用数
- 配额与实时用量

### 7.4 触顶后的表现

达到配额上限后：

- 已有简历、岗位、历史报告不会丢失
- 只会拒绝新的写入动作
- 接口通常返回 `402 Payment Required`

常见场景：

- 上传简历达到上限
- 创建 HR / 管理员账号达到上限
- 创建岗位达到上限
- 发起语音面试但未开通 `interview.voice`

### 7.5 续期与升级

客户侧不需要迁移数据，只需重新上传新的 `.lic` 文件即可。

## 八、常见问题

| 现象 | 处理方式 |
|---|---|
| 前端打开是 502 | 先看 `docker compose ps`，通常是 backend 未启动成功 |
| 简历一直停在 `pending` 或 `parsing` | 查看 `docker compose logs -f worker`，确认 worker 和 broker 状态 |
| LLM 调用超时 | 检查 `系统 → LLM 配置` 中 base URL 是否可达 |
| 发起语音面试返回 402 | 当前 license 未启用 `interview.voice` |
| 报告导出失败或按钮不可用 | 检查 license 是否启用 `report.export` |
| 候选人链接无法继续答题 | 邀请过期、被重发、被取消，或 session 过期；重新验证或重新发送邀请 |
| 候选人答题中途跳回邀请页 | session 过期或失效，需要重新验证 |
| 登录后看不到某些菜单 | 当前角色没有相应权限，联系管理员调整角色或组织范围 |
| 报告 / 面试详情返回 403 或 404 | 可能是权限不足，也可能是不在当前组织范围内 |
| `.lic` 上传提示机器指纹不匹配 | `MACHINE_FINGERPRINT_OVERRIDE` 与签发时不一致，需要重新申请 |
| IMAP 同步失败 | 检查邮箱地址、端口、app password、防火墙 |
| SMTP 发信失败 | 检查发件地址、认证信息、TLS 设置与网络可达性 |

## 九、合规与支持

### 9.1 合规边界

- 默认情况下，简历、候选人信息、面试问答都保存在客户自己的环境中
- 若客户主动配置公网 LLM，数据会发送给对应模型供应商
- License 校验是离线完成的，不依赖向供应商回传遥测

### 9.2 联系支持时建议提供

- `docker compose logs --since 1h backend frontend worker`
- 脱敏后的 `.env`
- License 页面截图
- `curl http://localhost/api/healthz?detail=1` 输出
- 如已启用诊断指标：`curl http://localhost/api/metrics` 关键片段
